Lộ trình implementation 18 ngày

Bỏ ngày đọc tổng quan riêng. Mỗi ngày phải tạo hoặc hoàn thiện một phần chạy được của CourseFlow Mini. Đọc code, đọc Docker, đọc Liquibase chỉ làm khi cần để implement.

Java 21 Spring Boot PostgreSQL Liquibase Keycloak Gateway Eureka MinIO Debezium Elasticsearch MongoDB Prometheus Grafana AI-assisted

Sơ đồ hệ thống

Bắt đầu từ System Overview để nhìn toàn bộ web/app client, Keycloak, gateway, service, database, event, chat và search trước khi bắt đầu bài.

Chuẩn trước khi vào ngày 01

  • Pull source, cài JDK 21, Maven, Docker, Node.
  • Chạy stack: cd backend/infra/docker && docker compose -f docker-compose.training.yml up -d --build.
  • Mở Keycloak: http://localhost:18080, admin console admin/admin.
  • Mở gateway health: http://localhost:28080/actuator/health.
  • Mở Kafka Connect status khi tới ngày 13: http://localhost:18083/connectors/courseflow-course-search-cdc/status.
  • Mở Prometheus/Grafana khi tới ngày 16 bằng profile observability: http://localhost:19090, http://localhost:13000.
  • Mở web admin http://localhost:5173/login, learner web http://localhost:3000/login; mobile app dùng cùng gateway API.

Service nền giữ nguyên

Service Dùng để học gì Cách dùng trong bài
discovery-service Eureka registry, service discovery, health/registration UI. Cho học viên mở http://localhost:8761, quan sát service đăng ký; không bắt tự code registry.
media-service Media metadata, signed URL, video readiness, MinIO bucket courseflow-media. Dùng như platform service khi làm course material/publish; không bắt implement lại trong 18 ngày.
MinIO S3-compatible object storage cho tài liệu, video source và file submission. Bắt học viên dùng signed URL, kiểm tra bucket trên console http://localhost:9001.
Debezium/Kafka Connect CDC từ bảng cf_course.public.courses sang Kafka topic để build search read model. Dùng ở ngày 13: cấu hình connector, đọc CDC envelope, verify search-service cập nhật Elasticsearch.
Prometheus/Grafana Metrics, scrape targets, dashboard, alert rule và service health. Dùng ở ngày 16: bật profile observability, đọc metric, truy lỗi bằng correlation id.

Format mỗi bài

Mục Nội dung cần có
Mục tiêu Feature cụ thể cần chạy được trong ngày.
Cách impl Service, API, DB migration, business rule, integration cần sửa.
Kết quả cần đạt Endpoint/web flow chạy được, response đúng contract, lỗi xử lý rõ.
Evidence Curl/browser flow, response chính, DB/log liên quan, kết luận pass/fail.

Đi từ tài liệu sang code

  1. Mở bài học trong thư mục training-roadmap/lessons để biết feature và API cần chạy.
  2. Mở api-contract.html để giữ đúng path cho admin web, learner web và mobile app.
  3. Tìm đúng comment trong code: rg 'TODO\(training-day-04' backend/services.
  4. Đọc thêm comment TRAINING(...) quanh controller, DTO, service và repository để hiểu boundary hiện có.
  5. Đề xuất DB/migration ngắn trước khi code: table, index, unique constraint, rollback note.
  6. Implement ở service layer trước; controller chỉ nhận request, gọi service và trả response.
  7. Chạy evidence cuối bài: happy path, một lỗi quan trọng, một read-back query hoặc log/DB check.

Khi nào coi là xong

  1. API chạy qua gateway, protected endpoint có auth/authz.
  2. DB thay đổi bằng Liquibase, không sửa tay database.
  3. Business rule nằm ở service layer, controller mỏng.
  4. Response đủ cho web/admin/learner dùng, lỗi có message rõ.
  5. Có evidence: request, response, DB/log, kết luận.
  6. Test tự động optional trong phase này.
