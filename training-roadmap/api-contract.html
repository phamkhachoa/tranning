Auth khi gọi API

Đường chính của mini stack là đăng nhập web/app qua Keycloak, sau đó gọi cùng API với Authorization: Bearer <access_token>. Gateway verify OIDC token, tự sinh internal JWT local cho downstream service, rồi authorization vẫn chạy đầy đủ ở gateway/access-control/domain service. Luồng đầy đủ nằm ở Auth Flow.

Khi viết curl evidence, có thể copy access token từ browser/app dev session sau khi login Keycloak. Nếu bật COURSEFLOW_TRAINING_AUTH_BYPASS=true, dùng header sau để giả lập user: 

X-Training-User-Id: 4
X-Training-User-Email: student@courseflow.local
X-Training-User-Roles: STUDENT

Với admin/instructor:

X-Training-User-Id: 2
X-Training-User-Email: instructor@courseflow.local
X-Training-User-Roles: INSTRUCTOR

Error contract

Mọi service nên trả cùng shape để web/app xử lý lỗi nhất quán:

{
  "statusCode": "409 CONFLICT",
  "title": "Conflict",
  "detail": "Course slug already exists",
  "errorCode": "COURSE_SLUG_DUPLICATED",
  "fieldErrors": []
}
  • 400: validation hoặc body sai kiểu.
  • 401: thiếu/invalid token.
  • 403: có login nhưng không đủ quyền.
  • 404: resource không tồn tại hoặc không được phép biết nó tồn tại.
  • 409: duplicate, invalid state transition, idempotency payload conflict.
  • 429: rate limit.
  • 502/503: downstream dependency lỗi hoặc timeout.

Quy ước path cần giữ thống nhất

  • /api/v1/**: learner/public API cho web learn và mobile app.
  • /api/admin/v1/**: backoffice API cho admin web, instructor/admin operations.
  • /internal/** và /backoffice/**: path trong service sau khi gateway rewrite, không gọi trực tiếp từ client.
  • POST /api/admin/v1/users là chuẩn để admin tạo learner. Learner /register là public self-registration qua Keycloak, không thay thế admin API.

Chat STOMP training auth

Với WebSocket/STOMP trên web hoặc app, gửi native header trong CONNECT frame:

Authorization: Training 4:student@courseflow.local:STUDENT

Destination mẫu:

CONNECT /ws/chat
SUBSCRIBE /topic/courses/{courseId}/chat
SEND /app/courses/{courseId}/send
