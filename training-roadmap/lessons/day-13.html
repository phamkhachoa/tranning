Ngày 13: Search CDC, suggest và recommend-lite

Dùng Debezium CDC để đồng bộ course published sang Elasticsearch, rồi search/autocomplete/recommend thay vì SQL LIKE hoặc DB polling.

Mục tiêu

  • Index course public vào Elasticsearch.
  • Debezium connector chạy RUNNING và emit CDC từ bảng public.courses.
  • Search keyword/filter/paging qua Elasticsearch.
  • Suggest và recommendations chạy nhanh cho UI.

Cách impl

  1. Đọc docker-compose.training.yml: Postgres chạy wal_level=logical, Kafka Connect expose 18083, setup container đăng ký connector sau khi course-service healthy.
  2. Đọc infra/docker/debezium/course-search-cdc-connector-config.json: connector class PostgreSQL, database cf_course, table public.courses, topic courseflow.course.public.courses.
  3. Start stack và verify connector bằng curl http://localhost:18083/connectors/courseflow-course-search-cdc/status; pass khi connector và task đều RUNNING.
  4. Đọc Kafka topic CDC bằng console consumer nếu cần debug: payload Debezium có op, before, after, source.lsn, ts_ms.
  5. Đọc CourseSearchEventConsumer: consumer group search-service, topic config courseflow.search.kafka.course-cdc-topic.
  6. Map Debezium row sang search DTO: id, code, title, slug, summary, departmentId, level, status. Nếu thiếu field bắt buộc thì log warn và mark processed.
  7. Idempotency key = courseId + op + tx/lsn; ES upsert/delete idempotent, processed marker lưu sau khi ES write xong.
  8. Rule projection: op=d hoặc status khác PUBLISHED thì delete document; status PUBLISHED thì upsert document.
  9. Đọc mapping Elasticsearch CourseSearchDocument: text fields dùng analyzer, exact fields keyword, status luôn keyword để filter.
  10. Implement search query: bool filter status=PUBLISHED, must multi_match title^4/summary khi có q, pageable có max size.
  11. Implement suggest: bool-prefix hoặc prefix-friendly query, limit max 10, trả DTO ngắn đủ cho UI autocomplete.
  12. Implement recommend-lite: keyword/level/departmentId thành should clauses có boost; nếu không có signal thì trả catalog published nổi bật hoặc recent.
  13. Giữ manual index endpoint staff-only cho backfill/debug; không dùng nó làm đường sync chính trong demo.
  14. Evidence bắt buộc: connector status RUNNING, publish course -> CDC topic có event -> ES có document -> search thấy course, archive/draft -> search không thấy.

Kết quả cần đạt

  • Search "java" trả course phù hợp.
  • Suggest "jav" trả top kết quả ngắn.
  • Recommend dùng category/level/keyword signal.
  • Update course title/summary bên course-service tự phản ánh sang Elasticsearch qua Debezium.
  • Không dùng repository SQL LIKE cho public search.

Evidence

GET http://localhost:18083/connectors/courseflow-course-search-cdc/status
Expected: connector.state=RUNNING, tasks[0].state=RUNNING

POST /api/admin/v1/courses/{id}/publish
Expected: row public.courses.status=PUBLISHED, Debezium emit event

GET /api/v1/search/courses?q=java&level=BEGINNER
Expected: results từ Elasticsearch

GET /api/v1/search/courses/suggest?q=jav
Expected: top suggestions

Negative:
- draft/archive course không xuất hiện hoặc bị delete khỏi index
- tắt Elasticsearch thì search-service báo lỗi rõ
