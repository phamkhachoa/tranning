Ngày 08: Assignment submission và feedback

Learner nộp bài, file attachment đi qua signed URL/MinIO, instructor chấm điểm và phản hồi theo lifecycle rõ.

Assignment Submission MinIO Feedback

Sơ đồ impl

flowchart LR Instructor["Instructor
Admin web"] --> Assignment["create assignment"] Assignment --> AssignmentDB[("assignment DB")] Student["Learner web"] --> Submit["submit/resubmit"] MobileApp["Mobile app"] --> Submit Student --> Upload["request signed upload URL"] MobileApp --> Upload Upload --> MinIO[("MinIO
courseflow-submissions")] Submit --> EnrollCheck["must be enrolled"] EnrollCheck --> SubmissionDB[("submission")] Instructor --> Grade["grade + feedback"] Grade --> Ownership["must teach course"] Ownership --> SubmissionDB Grade --> GradeEvent["grade update
API/event to gradebook"]

Mục tiêu

  • Instructor tạo assignment cho course.
  • Learner đã enroll submit/resubmit theo rule.
  • Instructor grade submission và lưu feedback.

Cách impl

  1. Đọc assignment-service: assignment, submission, attachment grant, rubric, grading, learning access client, ObjectStorageClient.
  2. Chốt assignment lifecycle: DRAFT/PUBLISHED/ACTIVE, availableAt/dueAt/lockAt optional, maxAttempts, submissionTypes TEXT/URL/FILE.
  3. Instructor create assignment: lấy instructor từ CurrentUser, check course staff access, validate maxScore > 0, dueAt sau availableAt nếu có.
  4. Learner submit: check active enrollment qua course/enrollment access client, check assignment visible/open, check attempts còn lại.
  5. Text/link submission: validate không rỗng, URL hợp lệ nếu submissionUrl, sanitize/limit length.
  6. File submission: gọi POST /api/v1/assignments/{id}/attachments/upload-url, PUT file lên MinIO bằng signed URL, submit kèm storageKey/fileName/contentType/size.
  7. Khi nhận storageKey trong submit, verify grant thuộc assignment + student + chưa hết hạn; có thể gọi storage.exists để chắc object đã upload.
  8. Resubmit rule: nếu maxAttempts > current attempt và chưa locked; submission mới tăng attemptNo, không overwrite bản cũ.
  9. Instructor grading: check course staff, score trong 0..maxScore, apply late penalty nếu có, lưu feedback và gradedBy/gradedAt.
  10. Rubric optional: nếu có rubric criteria, validate tổng điểm rubric khớp rawScore hoặc không vượt maxScore.
  11. Sau grade, phát outbox/event hoặc gọi gradebook integration theo contract để ngày 10 aggregate được.
  12. Download attachment: owner hoặc course staff mới lấy presigned download URL; không expose public MinIO object key như URL cố định.
  13. Evidence bắt buộc: upload-url -> PUT MinIO -> submit file, instructor grade, download URL works, submit quá hạn/không enroll bị reject.

Kết quả cần đạt

  • Learner submit bài và xem submission của mình.
  • Instructor xem queue cần chấm theo course.
  • Score nằm trong range hợp lệ.
  • Submission đã GRADED không resubmit nếu rule không cho phép.

Evidence

POST /api/v1/assignments/{id}/submissions
Authorization: Bearer <student_token>
Body: {"content":"..."}
Expected: status SUBMITTED

File flow:
POST /api/v1/assignments/{id}/attachments/upload-url
PUT file lên MinIO bằng signed URL
POST /api/v1/assignments/{id}/submissions với storageKey đã upload

POST /api/admin/v1/submissions/{id}/grade
Authorization: Bearer <instructor_token>
Body: {"score":85,"feedback":"Good"}
Expected: status GRADED

Negative:
- student chưa enroll submit -> 403
- instructor khác course grade -> 403
