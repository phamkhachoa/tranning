Ngày 16: Observability

Biết service đang sống hay chết, request lỗi ở đâu, và metric nào chứng minh hệ thống ổn.

Actuator Prometheus Grafana Correlation ID

Sơ đồ impl

flowchart LR Client["Web/App"] --> GW["API Gateway
X-Correlation-Id"] GW --> Svc["Domain services
actuator/prometheus"] Svc --> Logs["Structured logs
correlationId"] Prom["Prometheus"] -->|scrape| GW Prom -->|scrape| Svc Grafana["Grafana"] --> Prom Alert["Alert rules"] --> Prom Debug["Debug one failing request"] --> Logs Debug --> Grafana

Mục tiêu

  • Chạy Prometheus/Grafana bằng profile observability của training compose.
  • Đọc được /actuator/health và /actuator/prometheus của gateway/service.
  • Truy một request bằng X-Correlation-Id từ gateway sang service log.
  • Biết viết một alert đơn giản cho service down hoặc HTTP 5xx tăng cao.

Cách impl

  1. Start stack có observability: docker compose -f docker-compose.training.yml --profile observability up -d --build.
  2. Mở Prometheus http://localhost:19090/targets, xác nhận gateway và các service có state UP.
  3. Mở Grafana http://localhost:13000 với admin/admin, kiểm tra datasource Prometheus.
  4. Đọc backend/infra/observability/prometheus-training.yml: job nào scrape gateway, job nào scrape service.
  5. Đọc CorrelationIdGatewayFilter: gateway tạo hoặc forward X-Correlation-Id.
  6. Đọc CorrelationIdFilter trong common-library: service đưa correlationId vào MDC để log dễ tìm.
  7. Gọi một API có header X-Correlation-Id: training-day-16-001, sau đó grep log gateway và service theo id này.
  8. Đọc ApiExceptionHandler: lỗi phải có error shape thống nhất, không trả stack trace ra client.
  9. Thêm metric hoặc counter nhỏ nếu cần: số request bị reject do auth, số outbox publish fail, số search projection fail.
  10. Viết một alert training: service scrape down trong 2 phút hoặc HTTP server 5xx rate tăng.
  11. Không gắn label có cardinality cao như userId, email, token, full URL query vào metric.
  12. Evidence phải có ảnh/response Prometheus target UP, query metric, một log có correlationId, và một alert rule đọc được.

Kết quả cần đạt

  • Prometheus thấy đủ target chính.
  • Grafana đọc được datasource Prometheus.
  • Một request lỗi có thể truy từ response correlation id sang log.
  • Giải thích được khác nhau giữa health check, metric và log.
