Ngày 05: Publish workflow

Course đi qua trạng thái rõ ràng trước khi xuất hiện ở catalog public.

State Machine Permission Media readiness History

Sơ đồ impl

stateDiagram-v2 [*] --> DRAFT DRAFT --> IN_REVIEW: instructor submit-review IN_REVIEW --> CHANGES_REQUESTED: reviewer request changes CHANGES_REQUESTED --> DRAFT: instructor update IN_REVIEW --> PUBLISHED: admin/reviewer publish PUBLISHED --> ARCHIVED: admin archive ARCHIVED --> [*] DRAFT --> DRAFT: update draft fields PUBLISHED --> PUBLISHED: block direct important edits

Với video/document item, publish workflow gọi media-service để kiểm tra asset READY trước khi public course.

Mục tiêu

  • Course có transition: DRAFT -> IN_REVIEW -> PUBLISHED -> ARCHIVED.
  • Instructor submit review; admin/reviewer publish.
  • Publish bị chặn nếu module item tham chiếu video/document media chưa READY.
  • Mỗi transition có history/audit tối thiểu.

Cách impl

  1. Đọc state hiện tại trong course-service: course.status, reviewState, course_versions, review audit/history nếu có.
  2. Vẽ transition chính trước khi code: DRAFT -> IN_REVIEW -> APPROVED/PUBLISHED hoặc REJECTED -> DRAFT, PUBLISHED -> ARCHIVED.
  3. Implement submit-review: chỉ instructor/owner hoặc course staff; validate draft completeness trước khi đổi state.
  4. Readiness tối thiểu: title/summary/slug có dữ liệu, có ít nhất một module active, có ít nhất một item required, nếu item VIDEO thì videoMediaId không null.
  5. Media readiness: gọi GET /internal/media/videos/{videoId}/readiness bằng internal service token; video/document chưa READY thì submit/publish bị chặn với message rõ.
  6. Implement reviewer approve/publish: admin/reviewer độc lập, không cho cùng người vừa author vừa approve nếu muốn mô phỏng rule enterprise.
  7. Khi publish, tạo snapshot version immutable: curriculum, module order, item order, media references, pricing nếu có; learner runtime đọc snapshot thay vì draft live.
  8. Ghi audit transition: courseId, fromState, toState, actorId, reason/note, createdAt; dùng để review decision trail.
  9. Chặn transition sai thứ tự: publish DRAFT trực tiếp, submit archived course, update published snapshot bằng draft endpoint.
  10. Nếu publish thành công, status course phải thành PUBLISHED để Debezium Day 13 bắt event và search-service index.
  11. Nếu archive course, public catalog/search/enrollment phải không cho enroll mới; Debezium consumer sẽ delete khỏi search index ở Day 13.
  12. Evidence bắt buộc: submit thiếu media READY bị reject, submit đủ điều kiện OK, admin publish OK, public detail thấy course, transition sai trả 400/409.

Kết quả cần đạt

  • Course thiếu dữ liệu không submit review được.
  • Course IN_REVIEW publish thành PUBLISHED.
  • Catalog public chỉ thấy course PUBLISHED.
  • Transition sai thứ tự trả 409 hoặc 400.

Evidence

POST /api/admin/v1/courses/{id}/submit-review
Authorization: Bearer <instructor_token>
Expected: status IN_REVIEW

POST /api/admin/v1/courses/{id}/publish
Authorization: Bearer <admin_token>
Expected: status PUBLISHED

GET /api/v1/courses/{id}
Expected: public detail available

Negative:
- student publish -> 403
- publish DRAFT trực tiếp -> 409/400
