Ngày 04: Course CRUD

Implement CRUD đủ dùng cho admin/instructor và catalog public cho learner.

Mục tiêu

  • Instructor tạo và sửa course draft.
  • Course/module item có thể lưu mediaId lấy từ media-service, không tự upload file trong course-service.
  • Admin/instructor list course có filter/paging.
  • Public catalog chỉ hiển thị course published.

Cách impl

  1. Đọc course-service controller/service/entity/migration hiện có: courses, course_versions, course_modules, module_items, course_materials.
  2. Chốt aggregate boundary: course-service sở hữu course/module/item metadata; media-service sở hữu file/video metadata và MinIO object key; course chỉ lưu mediaId/videoMediaId.
  3. Thiết kế request tạo course draft: code, title, slug, summary, departmentId, level, listPrice/currency nếu dùng pricing; ownerId lấy từ CurrentUser, không nhận từ body.
  4. Migration/index cần có: unique code, unique slug, index status, index owner/department nếu admin list filter theo các field này.
  5. Implement create draft trong service layer transaction: validate role instructor/admin, validate department scope nếu có, normalize slug/code, set status DRAFT, reviewState DRAFT.
  6. Implement update draft: chỉ owner/staff được sửa, chỉ course mutable state mới sửa được; không cho update status bằng endpoint update thường.
  7. Implement module/item CRUD tối thiểu nếu lesson cần curriculum: create module, create item, reorder, archive item; dùng position unique theo module/course.
  8. Khi item là VIDEO/DOCUMENT, gọi media-service trước để có media/video id; course-service không upload file, không đọc MinIO bucket trực tiếp.
  9. Public catalog endpoint chỉ đọc PUBLISHED/published snapshot; admin endpoint được filter DRAFT/IN_REVIEW/PUBLISHED.
  10. DTO public phải loại bỏ field nội bộ như review note, draft-only data, internal audit nếu không cần cho learner.
  11. Xử lý lỗi: duplicate slug 409, update published trực tiếp 400/409, student create course 403, mediaId sai format 400.
  12. Evidence bắt buộc: instructor tạo draft, update title/summary, add module/item có mediaId, public catalog không thấy draft, admin list thấy draft.

Kết quả cần đạt

  • POST /api/admin/v1/courses tạo draft.
  • PATCH /api/admin/v1/courses/{id} sửa draft.
  • GET /api/admin/v1/courses?status=DRAFT có paging.
  • GET /api/v1/courses không trả draft.
