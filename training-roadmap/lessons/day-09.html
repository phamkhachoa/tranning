Ngày 09: Quiz attempt và auto scoring

Instructor tạo quiz, learner làm attempt, hệ thống chấm điểm tự động.

Quiz Attempt Auto Scoring

Sơ đồ impl

flowchart LR Instructor["Instructor
Admin web"] --> Author["create quiz/questions/options"] Author --> QuizDB[("quiz DB")] Learner["Learner web"] --> Start["start attempt"] MobileApp["Mobile app"] --> Start Start --> SafePayload["questions + options
no correct answer"] SafePayload --> Attempt[("attempt STARTED")] Learner --> Submit["submit answers"] MobileApp --> Submit Submit --> Score["auto scoring"] Score --> Result["score + passed + safe breakdown"] Result --> AttemptSubmitted[("attempt SUBMITTED")] Submit -->|already submitted| Conflict["409"]

Mục tiêu

  • Instructor tạo quiz với single choice/multiple choice.
  • Learner start attempt không thấy đáp án đúng.
  • Submit attempt trả score và passed flag.

Cách impl

  1. Đọc quiz-service: quiz, questions, options, attempts, answers, readiness endpoint cho course publish.
  2. Chốt quiz lifecycle: DRAFT -> PUBLISHED/ACTIVE -> ARCHIVED; course-service chỉ publish item quiz khi quiz readiness PUBLISHED/ACTIVE.
  3. Instructor create quiz: check course staff, validate title, timeLimit, maxAttempts, passScore nếu có.
  4. Question CRUD: hỗ trợ tối thiểu SINGLE_CHOICE/MULTIPLE_CHOICE; question text không rỗng, point > 0, ít nhất một correct option.
  5. Public learner view không được leak correct answer/explanation trước submit; admin view được thấy đáp án.
  6. Start attempt: check learner access/enrollment, quiz published, attempts còn lại, tạo attempt status IN_PROGRESS với startedAt.
  7. Submit attempt: validate attempt thuộc current user, status IN_PROGRESS, chưa hết time limit nếu rule bật.
  8. Scoring: single choice đúng full point; multiple choice có rule rõ full-match hoặc partial; normalize score scale và rounding.
  9. Sau submit, set status SUBMITTED/GRADED, submittedAt, score; answer detail lưu đủ để review nhưng learner chỉ thấy correct answers nếu reveal policy cho phép.
  10. Idempotency: submit lại attempt đã graded trả result cũ hoặc 409 theo rule, không double score.
  11. Emit grade event hoặc expose gradebook source endpoint để ngày 10 lấy điểm quiz.
  12. Evidence bắt buộc: instructor publish quiz, learner start/submit, score đúng, correct answer không leak trước submit, student2 không xem attempt student1.

Kết quả cần đạt

  • Quiz published mới cho learner làm.
  • Attempt không submit lại nếu rule không cho phép.
  • Score đúng với bộ answer mẫu.
  • Nested validation trả lỗi rõ khi input sai.

Evidence

POST /api/v1/quizzes/{quizId}/attempts
Authorization: Bearer <student_token>
Expected: attempt STARTED, không có correctAnswer

POST /api/v1/quizzes/attempts/{attemptId}/submit
Body: {"answers":[...]}
Expected: score, passed, breakdown an toàn

Negative:
- submit attempt đã SUBMITTED -> 409
- response start attempt leak đáp án -> fail review
