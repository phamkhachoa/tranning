Ngày 07: Learner progress và next action

Theo dõi tiến độ học theo learner/course và trả next action cho UI.

Progress Learner Runtime Read Model

Sơ đồ impl

flowchart LR LearnerWeb["Learner web"] --> Complete["complete lesson API"] MobileApp["Mobile app"] --> Complete Complete --> Access["must be enrolled"] Access --> Curriculum["course-service
modules + lessons"] Access --> ProgressDB[("progress rows")] ProgressDB --> Calc["completed / total"] Curriculum --> Calc Calc --> Dashboard["learning dashboard
progressPercent + nextAction"] Complete -->|same lesson twice| Idempotent["no double count"]

Mục tiêu

  • Learner xem được danh sách course đang học.
  • Mark lesson complete không làm progress vượt 100%.
  • API trả next action ổn định cho web/app.

Cách impl

  1. Đọc learner runtime trong course-service/enrollment-service: player, module progress, item progress, next action, learning access client.
  2. Chốt source of truth: enrollment xác nhận learner có quyền học course; course-service tính curriculum/progress dựa trên published snapshot/item progress.
  3. Implement guard trước khi mark complete: user phải có active enrollment hoặc course staff preview; không cho student khác ghi progress hộ.
  4. API mark item complete phải nhận courseId/moduleId/itemId; studentId lấy từ CurrentUser; server tự set completedAt.
  5. Validate item thuộc course published snapshot hiện tại; item archived/draft không được mark complete.
  6. Progress percent = completed required items / total required items; optional item không bắt buộc nhưng có thể hiển thị completed riêng.
  7. Next action chọn item required đầu tiên chưa complete theo module position/item position và prerequisite nếu có.
  8. Nếu complete lại item đã complete, endpoint idempotent: không tạo duplicate row, không giảm progress.
  9. Nếu course publish version mới, xác định rule: progress theo publishedVersionNo hiện tại hoặc migrate mapping; trong training ghi rõ dùng current published snapshot.
  10. Expose read endpoint cho learner: progress summary, module states, item states, next action; không trả progress của learner khác.
  11. Có thể emit event learning.item.completed hoặc cập nhật enrollment progress summary nếu code đã có outbox.
  12. Evidence bắt buộc: student mark complete, progress tăng, next action đổi, mark complete item không thuộc course trả 404/400, student2 không thấy progress student1.

Kết quả cần đạt

  • GET /api/v1/enrollments hoặc dashboard trả progress percent.
  • Complete lesson trùng không tăng sai.
  • Course không có lesson trả progress hợp lý theo rule đã chốt.
  • Student không xem được progress của student khác.

Evidence

POST /api/v1/progress/lessons/{lessonId}/complete
Authorization: Bearer <student_token>
Expected: progress tăng đúng

GET /api/v1/enrollments
Expected: course có progressPercent và nextAction

Negative:
- lesson thuộc course chưa enroll -> 403
- complete lại lesson đã complete -> idempotent
