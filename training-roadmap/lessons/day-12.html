Ngày 12: Course chat realtime

Thêm chat theo course để học REST history, MongoDB và STOMP realtime.

Chat MongoDB WebSocket/STOMP

Sơ đồ impl

flowchart LR Client["Learner web"] --> Connect["STOMP CONNECT"] MobileApp["Mobile app"] --> Connect Connect --> Auth["Authorization header"] Auth --> ChatSvc["chat-service interceptor"] ChatSvc --> CourseAccess["course access check"] CourseAccess --> Subscribe["SUBSCRIBE /topic/courses/{id}/chat"] Client --> Send["SEND /app/courses/{id}/send"] MobileApp --> Send Send --> Validate["message validation"] Validate --> Mongo[("MongoDB message history")] Validate --> Topic["broadcast topic"] Topic --> Client Topic --> MobileApp

Mục tiêu

  • Mỗi course có room chat.
  • Learner/instructor có access course mới gửi/đọc message.
  • REST history và STOMP realtime chạy được ở mức demo.

Cách impl

  1. Đọc chat-service: REST controller, STOMP/WebSocket config, auth interceptor, MongoDB message document.
  2. Chốt room model: một course có một room chính; roomId có thể là courseId hoặc document riêng chứa courseId/members.
  3. REST get/create room: user phải có course access; staff và enrolled learner được vào, user ngoài course bị 403.
  4. STOMP connect phải verify token/header training; principal chứa userId/email/roles, không tin userId từ message body.
  5. Destination rule: SEND chỉ được tới /app/courses/{courseId}/send, SUBSCRIBE chỉ tới topic course user được phép.
  6. Send message: validate content không rỗng, limit length, trim, optional sanitize; server set senderId/senderName/createdAt.
  7. Persist MongoDB trước hoặc cùng lúc broadcast; nếu persist fail thì không báo delivered giả.
  8. History endpoint: paging bằng before/limit, sort createdAt desc/asc rõ, limit max để tránh tải quá nhiều message.
  9. Read state optional: lưu lastReadAt per user/course nếu có thời gian; nếu chưa thì chỉ cần history + realtime.
  10. Moderation tối thiểu: instructor/admin có thể delete/hide message hoặc phase sau; nếu chưa làm, ghi backlog.
  11. Web/app evidence: mở hai user/tab hoặc một web một app client, subscribe cùng course, gửi message thấy realtime và refresh vẫn có history.
  12. Negative evidence: student chưa enroll subscribe/send bị reject; STOMP không token bị reject.

Kết quả cần đạt

  • Message lưu vào MongoDB theo courseId + createdAt.
  • History trả paging/limit ổn định.
  • STOMP topic /topic/courses/{courseId}/chat nhận message mới.
  • User chưa enroll không đọc/gửi được.

Evidence

GET /api/v1/chat/courses/{courseId}/messages
Authorization: Bearer <student_token>
Expected: history

STOMP CONNECT Authorization: Training 4:student@courseflow.local:STUDENT
SUBSCRIBE /topic/courses/{courseId}/chat
SEND /app/courses/{courseId}/send
Expected: subscriber nhận message

Negative:
- user chưa enroll -> 403/MessagingException
